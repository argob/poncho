Datos personales

Usar este encabezado para agrupar los datos y separarlos de otros para una lectura más cómoda.

CUIL

Sirve para identificar a un ciudadano. Es más preciso que el DNI. Si se pide este dato no hace falta pedir más datos personales de identificación.

Funcionamiento

Acepta puntos, guiónes y barras, pero envía sólo números. Valida un mínimo de caracteres numéricos.

Nombres

Sirve para identificar y llamar al usuario por su nombre en páginas donde esté loggeado o correos electrónicos que se le envíen.

Apellidos

Sirve para identificar al usuario. No pedir si no es necesario identificar al ciudadano (por ejemplo un formulario de sugerencias donde con un nombre y un correo electrónico alcanza.

Sexo

Sirve para identificar al usuario. Hay servicios públicos y/o médicos que requieren de identificar el sexo del usuario. En general no hace falta pedirlo.

Fecha de nacimiento

Sirve para saber la edad del usuario. Sirve para servicios que requieran cierta mayoría o rango de edad.

Funcionamiento

Acepta números y barras. El formato de datos enviado es dd-mm-aaaa. Esto es siempre un único campo input date. Si a la base de datos a donde hay que guardar esto tiene los campos día, mes y año por separado, se debe parsear en el envío.

Otros datos personales

Forman parte de los datos personales, pero no son necesarios para identificar al ciudadano.

Detalle de elementos

No usar encabezado

Este grupo de datos va debajo de los datos personales (y por ende agrupado bajo el encabezado de datos personales).

Tipo de documento

Sirve para identificar a un ciudadano. Es menos preciso que el CUIL ya que pueden haber DNI duplicados. No pedir si el servicio no necesita este dato para funcionar. Siempre se debe intentar usar CUIL primero.

Funcionamiento

Va en la misma fila a la izquierda de Número de documento.

Número de documento

Complementa al dato Tipo de documento.

Funcionamiento

Acepta números y barras. Acepta puntos, guiónes y barras, pero envía sólo números. Va en la misma fila a la derecha de Tipo de documento.

Estado Civil

Sirve para saber el estado civil del ciudadano. No pedir si el servicio no necesita este dato para funcionar.

País de nacimiento

Sirve para saber la nacionalidad original del ciudadano. No pedir si el servicio no necesita este dato para funcionar.

Funcionamiento

Acepta números y barras. La primera opción del listado es Argentina.
